# 数据分析控制台业务场景与测试计划 (Test Plan)

本文档基于数据分析控制台（BI 报表引擎）的“端口与适配器（六边形）架构”与“Vue 3 组合式架构”，梳理了系统的核心结构与 6 大关键业务场景，并为每个场景提供了全面的验收测试用例（Checklist），可作为手动测试或编写自动化测试用例的直接依据。

---

## 一、 系统架构梳理

### 1. 后端架构 (Python 3.9 + FastAPI)
* **`core/` (领域核心层)**: 不依赖框架的纯领域模型（`models.py` 的 DTOs）和抽象接口（`ports.py` 的 DataSourcePort、RepositoryPort）。
* **`infrastructure/` (基础设施层)**: 数据库持久化实现。已**全面剥离物理外键**，依靠 `repositories.py` 中的逻辑接管数据聚合拼接，使用 SQLAlchemy 构建 `orm_models.py`。
* **`adapters/` (适配器层)**: 具体的数据库方言适配器（如 `clickhouse.py`, `duckdb.py`），将抽象指令转为物理查询。
* **`services/` (服务层)**: `query.py` 提供 AST 语法树智能下钻、宏参数正则预编译替换以及 LRU 查询缓存。
* **`api/` (应用接入层)**: 瘦控制器（Thin Controllers），负责 HTTP 路由及依赖注入。

### 2. 前端架构 (Vue 3 + Vite)
* 核心业务视图集中在 `front-end/src/components/BiDashboard.vue`。
* 分为三大标签页域：**数据看板 (Dashboards)**、**分析工作台 (Queries)**、**配置中心 (Data Sources)**。
* 利用 `shallowReactive` 和 `markRaw` 彻底切断了大数组的深层 Proxy 开销，确保了拖拽引擎 (`vue3-grid-layout`) 的极致流畅。

## 二、 全面测试用例清单

### 场景 1：多数据源管理 (Data Source Management)
**目标**：验证能否正确接管 ClickHouse、DuckDB 等类型的数据源连接参数及连通性测试。

- [ ] **TC-1.1 新建有效数据源**
  - **操作**：填入合法的凭证，点击保存。
  - **预期**：后端 `test_connection()` 返回 True，数据源保存成功，并在左侧列表展现。
- [ ] **TC-1.2 新建无效数据源拦截**
  - **操作**：填入错误的端口或密码，点击保存。
  - **预期**：后端连接失败，API 返回 400，前端提示“Failed to connect”，数据不入库。
- [ ] **TC-1.3 密码安全编辑逻辑**
  - **操作**：编辑已存在的数据源，保持密码框为空，仅修改 `host` 等其他字段，保存。
  - **预期**：后端自动隐式继承原密码进行连通性测试，保存成功且不覆盖/破坏原有密码。
- [ ] **TC-1.4 物理表字典探索**
  - **操作**：在分析工作台左侧下拉框切换不同数据源。
  - **预期**：正确触发 `get_tables` 接口，树形菜单渲染出该数据源下所有的表名。

### 场景 2：SQL 工作台与组件固化 (Query Editor & Persistence)
**目标**：验证 SQL 执行、图表渲染、宏替换与状态保存是否稳定。

- [ ] **TC-2.1 基本 SQL 与图表渲染**
  - **操作**：输入标准的聚合 SQL，执行并切换预览类型为 Table / Bar / Line / Pie。
  - **预期**：数据正常返回，不同类型的 ECharts 及表格能够根据 `columns` 和 `data` 正确渲染结构。
- [ ] **TC-2.2 宏参数预编译 (Macro Variables)**
  - **操作**：在 SQL 中写入 `{{region}}`，在工作台宏参数栏添加 `key: region, value: 'CN'` 并执行。
  - **预期**：正则安全拦截器生效，成功下发替换后的合法 SQL 给数据库。
- [ ] **TC-2.3 恶意宏参数防注入 (Security)**
  - **操作**：宏参数 `value` 输入带有单引号的恶意注入字符串（如 `' OR 1=1 --`）。
  - **预期**：触发后端防注入非法字符报错，拒绝执行查询。
- [ ] **TC-2.4 查询版本保存与更新**
  - **操作**：对查询执行“保存”，微调 SQL 后执行“更新当前查询”或“另存为新查询”。
  - **预期**：Repository 成功落盘，左侧“已保存的查询”列表动态刷新，看板中引用的该图表随之更新。

### 场景 3：条件格式与高亮阈值 (Threshold Formatting)
**目标**：验证前端渲染引擎对动态高亮规则的响应和颜色智能自适应计算。

- [ ] **TC-3.1 基本条件高亮匹配**
  - **操作**：给指标列添加 `> 100` 标红，`< 50` 标绿的规则。
  - **预期**：表格视图对应单元格背景色渲染准确。
- [ ] **TC-3.2 区间匹配 (Between/Not Between)**
  - **操作**：设置 `between 50 ~ 100` 显示黄色。
  - **预期**：介于区间的单元格变黄。
- [ ] **TC-3.3 字体颜色智能反转 (Luminance heuristics)**
  - **操作**：将背景色设置为极深色（透明度 Alpha > 0.6）。
  - **预期**：触发前端自动推算，该单元格文字颜色自动反转为白色 (`#ffffff`) 以保证对比度和可读性。

### 场景 4：动态大屏看板画布 (Dashboard & Layout)
**目标**：验证关联查询脱离外键后的稳定性、拖拽响应性能、以及全局宏参数合并逻辑。

- [ ] **TC-4.1 大屏增删改查与级联解耦**
  - **操作**：创建看板、推入组件、保存看板；随后删除看板，或删除看板内依赖的某个图表组件 (Saved Query)。
  - **预期**：Repository 层能够通过显式 SQL (如 `delete from dashboard_widgets`) 手动清除中间关联数据，不留脏数据且不报错。
- [ ] **TC-4.2 大数据量拖拽性能 (ShallowRef Test)**
  - **操作**：在看板加载包含数百列、千行的极宽表组件，进入“编辑模式”并拖拽调整位置大小。
  - **预期**：全程丝滑不掉帧，因为 `shallowReactive` 成功阻断了 Vue Proxy 的深层递归代理瓶颈。
- [ ] **TC-4.3 全局宏提取与局部覆盖 (Macro Scoping)**
  - **操作**：在画布放置多个带有 `{{version}}` 宏的图表，为其中一个单独配置“局部参数覆盖”，另一个不配。
  - **预期**：
    1. 看板顶部提取出 `version` 宏输入框。
    2. 在全局框输入值时，只有未被局部覆盖的图表会联动刷新。
    3. 局部配置过的图表不受全局控制干扰。

### 场景 5：AST 智能明细穿透 (Smart Drill-Through)
**目标**：验证底层引擎（基于 `sqlglot`）如何动态改写复杂 SQL 的投影进行对账。

- [ ] **TC-5.1 基础聚合穿透**
  - **操作**：点击任意普通的 `SUM()` 或 `AVG()` 指标结果单元格。
  - **预期**：底层抛弃聚合函数与 `GROUP BY`，只保留原始宽表以及 `WHERE`/`JOIN` 子句，成功弹出明细抽样。
- [ ] **TC-5.2 `COUNT(DISTINCT)` 智能去重穿透**
  - **操作**：点击形如 `COUNT(DISTINCT user_id)` 的去重指标。
  - **预期**：AST 正确捕获 Distinct 特征，将投影改写为类似 `SELECT table.* LIMIT 1 BY table.user_id` 的语法，实现去重展示且不丢失主表维度上下文。
- [ ] **TC-5.3 下钻分页与上下文维持**
  - **操作**：在明细弹窗中触发分页切换（如点击第 2 页）。
  - **预期**：请求成功携带原有的 Filters 和 Macros 字典，并追加正确的 `LIMIT` 和 `OFFSET` 进行下钻翻页。

### 场景 6：Excel 所见即所得导出 (Excel Export)
**目标**：验证数据与前端可视化样式能否同步输出为物理文件。

- [ ] **TC-6.1 图表数据导出**
  - **操作**：在工作台或看板图表右上角点击“导出 Excel”。
  - **预期**：成功调用 ExcelJS 下载包含对应列头与数据的 `.xlsx` 文件。
- [ ] **TC-6.2 带有阈值样式的导出 (Style preservation)**
  - **操作**：给图表配置阈值红绿灯，点击导出，打开下载的文件。
  - **预期**：Excel 单元格的背景色 (Pattern/ARGB) 和字体颜色与网页上看到的实时渲染效果完美 1:1 对齐同步。
