# 通用要求与规范 (Rules)

1. **语言要求**：后续所有的对话和回答都必须使用中文。
2. **代码版本控制**：每次任务执行完毕或完成一个逻辑闭环后，都必须进行 `git commit` 提交代码。
3. **文档维护**：每次执行任务完毕后，必须确认是否需要更新 `AGENTS.md`, `README.md` 以及其他可能涉及的 Markdown 说明文档，并保持系统设计文档的同步更新。
4. **数据库连接**：本机 ClickHouse 客户端一律采用 `clickhouse client` 命令进行连接和查询。
5. **元数据表管理**：任何涉及平台元数据（PostgreSQL）表结构的修改（如在 `meta_models.py` 中增删改列），必须同步更新 `metadata_schema.md` 数据字典文档，以确保架构设计信息不丢失。
6. **系统架构同步**：任何涉及系统基础部署拓扑、核心链路改动或模块职责划分的修改，都必须同步检查并更新 `4_plus_1_views.md` 中的 PlantUML 代码，以保障架构视图的绝对时效性。
